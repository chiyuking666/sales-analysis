# 销售统计整合工具

## 项目简介

这是一个使用 Python 编写的销售数据统计工具。

支持读取多个 Excel 文件，自动合并数据、清洗异常数据、统计销售情况，并生成汇总报表。

---

## 项目结构

```
sales_project/
│
├── input/            # 输入Excel文件
├── output/           # 输出统计结果
│
├── reader.py         # 读取Excel       
├── analyzer.py       # 数据清洗，数据统计
├── exporter.py       # 导出Excel
├── generate_data.py  # 生成测试用Excel数据
├── main.py           # 程序入口
│
└── README.md
```

## 使用方法

### 1. 准备输入数据

把所有销售 Excel 放入

```
input/
```

当前版本会读取 `input/` 下所有 `.xlsx` 文件，扩展名大小写不敏感，例如 `.xlsx`、`.XLSX` 都会识别。

程序会自动跳过 Excel 打开时产生的临时文件，例如 `~$sales_1.xlsx`。

如果你暂时没有可用的销售数据，可以先运行测试数据生成脚本：

```bash
python generate_data.py
```

该脚本会在 `input/` 目录下生成 10 个示例销售 Excel 文件，便于快速验证主流程。

### 2. 运行统计程序

运行

```bash
python main.py
```

程序将在

```
output/
```

生成

```
summary.xlsx
```

---

## 输入数据要求

Excel 至少包含以下字段：

| 列名 |
|----|
| 地区 |
| 产品 |
| 数量 |
| 单价 |

说明：

- 当前统计逻辑实际依赖 `地区`、`产品`、`数量`、`单价` 四列。
- `日期` 列不是当前版本的必需字段，也不会参与汇总计算。
- 当前版本只读取 `.xlsx` 文件，不处理 `.xls`、`.xlsm` 等其他 Excel 格式。

程序会自动计算销售额。

---

## 输出结果

summary.xlsx 包含多个工作表：

- 总销售表
- 地区销售统计
- 产品销售统计

---

## 数据清洗

程序会自动：

- 删除完全空白行
- 删除重复数据
- 将地区为空的数据修改为“未知”
- 转换数量、单价为数值类型
- 删除非法数据（数量≤0、单价≤0）

## 已知限制

- `产品` 为空的记录会保留在“总销售表”中，但不会出现在“产品销售统计”中，因此两张表的销售额合计可能不一致。
- `地区` 建议使用字符串。若填入数字或其他非字符串值，清洗后可能无法正确进入“地区销售统计”。
- 如果 `output/summary.xlsx` 正在被 Excel 打开，导出会失败。关闭该文件后重新运行即可。
